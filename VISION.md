# Reiser4-NX Vision

Most filesystems treat storage like a warehouse.

Bytes in.
Bytes out.
Everything else is somebody else’s problem.

Reiser4-NX aims for something different.

Not a filesystem that dictates meaning.
A filesystem that allows meaning to emerge.

A filesystem willing to become substrate.

One where researchers, artists, archivists, AI developers, scientists, preservationists, and complete lunatics can invent entirely new data formats, metadata systems, sidecar standards, indexing schemes, provenance layers, memory systems, and semantic structures without begging permission from giant corporations or waiting for standards bodies to wake from cryostasis.

Because then researchers can invent insane formats without waiting for:

- Microsoft
- Apple
- Adobe
- Linux kernel mainline
- SQLite cults
- cloud APIs

The filesystem becomes fertile soil instead of rigid law.

That is the dream.

Not a locked cathedral.
A frontier.

Directory-scoped schemas.
Filesystem-managed sidecars.
Portable metadata ecosystems.
Identity-aware storage.
Structured auxiliary knowledge.
Recoverable semantic layers.
Machine-readable and human-readable coexistence.

A filesystem where:

- comic researchers can attach panel extraction metadata
- archivists can preserve provenance chains
- AI datasets can carry lineage and embeddings
- photographers can preserve edit histories and perceptual hashes
- scientists can build insane experimental structures
- artists can create living media ecosystems

without fighting the storage layer itself.

The goal is not “more features.”

The goal is freedom.

Not cloud dependence.
Not app dependence.
Not vendor dependence.

Storage should not force imagination into tiny approved boxes.

A filesystem should be able to carry civilization’s weirdness gracefully.

Reiser4 was one of the few filesystems that ever seemed philosophically willing to attempt that.

Reiser4-NX exists because that idea deserves another chance.
