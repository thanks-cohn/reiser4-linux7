# Reiser4-Linux7 Failure Registry

Known smoke and runtime blockers are tracked here until a focused test proves them fixed.

| Entry | Symptom | Reproduction command | Expected breadcrumb | Artifact path pattern | Current status | Next target |
| --- | --- | --- | --- | --- | --- | --- |
| `MKDIR_EPERM_CURRENT_BLOCKER` | `mkdir` on mounted Reiser4 returns `EPERM` / operation not permitted. | `sudo ./tests/smoke_mkdir_basic.sh` | `SMOKE_MKDIR_BASIC_FAIL error="..."` | `artifacts/smoke_mkdir_basic-<timestamp>/` | Open current blocker; expected to fail until mkdir behavior is fixed. | Use BUMRUSH26_MKDIR/dmesg breadcrumbs to patch only the confirmed failing branch. |
| `TEARDOWN_MODULE_REF_STUCK_AFTER_FAILED_SMOKE` | After a failed smoke path, `rmmod reiser4` reports module in use or `lsmod` shows a stuck reference. | `sudo ./tests/smoke_failed_operation_teardown.sh` | `SMOKE_RMMOD_FAIL module_ref_stuck=1` or `SMOKE_TEARDOWN_AFTER_FAILURE_FAIL` | `artifacts/smoke_failed_operation_teardown-<timestamp>/` | Open teardown risk after failed operations. | Compare before/after/cleanup state and fix leaked references. |
| `LOOP_DEVICE_DELETED_IMAGE_STUCK` | `losetup -a` shows a Reiser4 test image, including deleted images, still attached after cleanup. | `sudo ./tests/smoke_failed_operation_teardown.sh` | `SMOKE_LOOP_STUCK` or `SMOKE_LOOP_STUCK_DELETED_IMAGE` | `artifacts/<smoke-test>-<timestamp>/state-after-cleanup.txt` | Open cleanup symptom observed after failed tests. | Identify whether failed unmount prevents loop autoclear; fix root filesystem state first. |
| `KTXNMGRD_STUCK_AFTER_FAILURE` | `ps -ef` shows `ktxnmgrd` after smoke cleanup. | `sudo ./tests/smoke_failed_operation_teardown.sh` | `SMOKE_KTXNMGRD_STUCK` | `artifacts/smoke_failed_operation_teardown-<timestamp>/state-after-cleanup.txt` | Open daemon/thread shutdown risk. | Trace transaction/thread lifetime after failed operations. |
| `INSMOD_FILE_EXISTS_WHEN_MODULE_PRELOADED` | `insmod ./reiser4.ko` fails with file exists because the module is already loaded. | `sudo ./tests/smoke_module_lifecycle.sh` with preloaded `reiser4` | `SMOKE_INSMOD_FAIL` or `SMOKE_PREFLIGHT_FAIL module_preloaded=1` | `artifacts/smoke_module_lifecycle-<timestamp>/` | Guarded by preflight; clean state required. | Reboot or unload the stuck module before running foundational tests. |
| `SMOKE_SCRIPT_HIDES_FAILURE_LINE_ONLY` | Old scripts could hide the real failure behind a generic line-number failure. | Any old smoke path that exits with only `command failed at line X`. | V3 replacement emits stage-specific `SMOKE_*_FAIL error="..."`. | `artifacts/<smoke-test>-<timestamp>/command-log.txt` | Addressed by V3 breadcrumb/artifact harness; monitor regressions. | Keep adding explicit breadcrumbs for each new smoke stage. |

The registry intentionally does not claim V3 personal readiness. A blocker remains open until the corresponding V3 smoke test passes from a clean state and its summary records clean cleanup state.
