# Reiser4-NX Smoke Tests

These smoke tests are intentionally staged and breadcrumb-heavy. They are not a V1 claim; they preserve the exact state around the current mkdir and teardown blockers.

| Test | Purpose | Pass means | Fail means | Current expected result | Artifacts produced | Next debugging target |
| --- | --- | --- | --- | --- | --- | --- |
| `tests/smoke_build_module.sh` | Prove kernel-header availability, clean Kbuild invocation, and `reiser4.ko` artifact health. | The module can be cleaned, rebuilt, and found at `./reiser4.ko`; BUMRUSH26 instrumentation presence/absence is reported. | Build environment, headers, Kbuild, or module artifact generation is broken before runtime testing. | Should pass on a host with matching kernel headers. | `artifacts/smoke_build_module-<timestamp>/summary.txt`, make logs, state, dmesg snapshots. | Fix Kbuild/header mismatch or compiler/module artifact failure. |
| `tests/smoke_module_lifecycle.sh` | Isolate pure module insertion/removal with no mounts. | `insmod` works, `lsmod` sees `reiser4`, `/proc/filesystems` registers it, and `rmmod` unloads it. | A failure here is a module lifecycle problem, not a filesystem operation problem. | Should pass only on a clean boot or clean host with no preloaded/stuck `reiser4`. | `artifacts/smoke_module_lifecycle-<timestamp>/summary.txt`, before/after state, before/after dmesg. | Investigate preloaded module, registration failure, or unload references independent of VFS mutation. |
| `tests/smoke_mkfs_mount_unmount.sh` | Prove mkfs, mount, root stat, unmount, and unload without mutation. | A formatted loop image mounts as Reiser4, root stat works, unmount succeeds, and the module unloads. | Basic format/mount/unmount path is broken before file operations. | mkfs/mount/root stat likely pass on a capable host. | `artifacts/smoke_mkfs_mount_unmount-<timestamp>/summary.txt`, mkfs log, root stat, state, dmesg. | Debug mkfs, mount option, superblock, root inode, unmount, or module refcount. |
| `tests/smoke_regular_file_rw.sh` | Prove regular file create/write/read/sync/remount/read without mkdir. | Root-level regular file IO survives sync, unmount, remount, and readback. | Regular file mutation or persistence is broken independent of directory creation. | May pass depending on current code and teardown state. | `artifacts/smoke_regular_file_rw-<timestamp>/summary.txt`, mkfs/read logs, state, dmesg. | Debug regular-file create, writeback, sync, remount persistence, or cleanup. |
| `tests/smoke_mkdir_only.sh` | Microscope for the current mkdir blocker. | `mkdir` under the mounted image succeeds and cleanup state remains visible. | `mkdir` failed; exact errno/message, state, dmesg, and BUMRUSH26_MKDIR breadcrumbs are preserved. | Currently expected to fail with `EPERM`. | `artifacts/smoke_mkdir_only-<timestamp>/summary.txt`, `dmesg-after-mkdir-failure.txt`, filtered dmesg, `bumrush26-mkdir.txt`, state snapshots. | Patch the confirmed `BUMRUSH26_MKDIR` returning branch; do not guess. |
| `tests/smoke_teardown_after_failure.sh` | Intentionally trigger the known mkdir failure and prove whether teardown survives it. | The expected mkdir failure occurs, then unmount/loop detach/rmmod leave no module, ktxnmgrd/entd, or deleted-image loop residue. | Failed mkdir poisons teardown or leaves a stuck module, kernel thread, or deleted loop image. | Currently expected to expose stuck module/ktxnmgrd risk. | `artifacts/smoke_teardown_after_failure-<timestamp>/summary.txt`, mkdir-failure state/dmesg, cleanup state/dmesg. | Fix teardown/refcount/thread shutdown after failed operations. |
| `tests/smoke_reiser4_v1.sh` | Full staged V1 smoke with preflight, mkfs, insmod, mount, root stat, file IO, mkdir, optional rename/delete, remount verify, and teardown. | Every stage, including mkdir and teardown, is clean. | V1 remains blocked; `summary.txt` names the failed stage and exact breadcrumb. | Not passed until mkdir and teardown are clean. | `artifacts/smoke_reiser4_v1-<timestamp>/summary.txt`, mkfs/root stat/read logs, failure state, pre/post cleanup dmesg. | Use smaller smoke failures first; only use this after build/lifecycle/mount/file tests pass. |
| `tests/run_smoke_suite.sh` | Run the staged smoke suite in dependency order and stop before invalid downstream tests. | Foundational tests pass; known mkdir blocker is reported; V1 only runs when prerequisites are meaningful. | The suite stops at the first foundational failure or reports teardown/V1 not passed. | Should stop early on foundational failures; mkdir failure is a known blocker, not a V1 pass. | `artifacts/run_smoke_suite-<timestamp>/summary.txt`, per-test captured logs, suite state/dmesg. | Follow the first failing foundational stage before chasing downstream symptoms. |

## Breadcrumb contract

All scripts print `REISER4_SMOKE_BEGIN`, stable `SMOKE_*_PASS` / `SMOKE_*_FAIL` breadcrumbs, `SMOKE_ARTIFACTS_AT`, and `REISER4_SMOKE_END`. Every script writes `summary.txt`, saves dmesg before cleanup and after cleanup, saves state before test, after failure, and after cleanup, and exits nonzero for real failures.
