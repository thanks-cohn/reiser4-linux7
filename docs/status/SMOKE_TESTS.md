# Reiser4-NX V3 Personal Readiness Smoke Tests

These smoke tests are staged, artifact-heavy, and designed to report exact state with stable `SMOKE_*` breadcrumbs. They do not claim V1, V3, or production readiness. V3 personal status is not `READY_TO_TRY` until all required tests pass cleanly.

| Test | Purpose | Pass means | Fail means | Required logs | Artifacts | Current expected result |
| --- | --- | --- | --- | --- | --- | --- |
| `smoke_build_module.sh` | Build module against current kernel headers | clean Kbuild invocation creates reiser4.ko | headers/compiler/Kbuild/module artifact blocked | kernel/gcc/git/warnings/module size | `artifacts/smoke_build_module-<timestamp>/` | Likely expected to pass on Ubuntu 24.04 LTS / Linux 6.8 with matching headers |
| `smoke_module_lifecycle.sh` | Pure insmod, procfs registration, rmmod | module loads, /proc/filesystems lists reiser4, module unloads | preloaded module, registration, or refcount issue | lsmod before/after, /proc/filesystems, dmesg danger scan | `artifacts/smoke_module_lifecycle-<timestamp>/` | Expected to pass only from clean boot or clean module state |
| `smoke_mkfs_image.sh` | Fresh loopback mkfs.reiser4 image | mkfs.reiser4 exits 0 | mkfs/reiser4progs/image setup blocked | mkfs version, image size, block/UUID when visible, exit code | `artifacts/smoke_mkfs_image-<timestamp>/` | Expected to pass |
| `smoke_mount_root_stat_unmount.sh` | Mount root, stat root, unmount, rmmod | mount/stat/unmount/rmmod all pass | basic mount or cleanup blocked | mount options, stat, findmnt, lsmod cleanup | `artifacts/smoke_mount_root_stat_unmount-<timestamp>/` | Expected to pass from clean state |
| `smoke_regular_file_create.sh` | Create a normal root-level file | file create/stat succeeds | regular file create blocked | path, inode, stat, dmesg scan | `artifacts/smoke_regular_file_create-<timestamp>/` | May pass |
| `smoke_regular_file_write_read.sh` | Write known bytes and read exact bytes | sha256 before/after matches | regular file write/read blocked | sha256 before/after, size, sync result | `artifacts/smoke_regular_file_write_read-<timestamp>/` | May pass |
| `smoke_regular_file_remount_verify.sh` | Verify file content after remount | hash matches after remount | persistence/remount blocked | sha256 before/after remount, remount result | `artifacts/smoke_regular_file_remount_verify-<timestamp>/` | May pass |
| `smoke_rename_file.sh` | Rename file and verify paths | old path gone, new path readable | rename operation blocked | old/new path, stat output | `artifacts/smoke_rename_file-<timestamp>/` | May pass |
| `smoke_delete_file.sh` | Delete file and verify after remount | file remains gone | unlink/remount persistence blocked | unlink result, remount verification | `artifacts/smoke_delete_file-<timestamp>/` | May pass |
| `smoke_mkdir_basic.sh` | Create one directory | mkdir succeeds | known EPERM blocker hit | BUMRUSH26_MKDIR when present, stat dir, dmesg scan | `artifacts/smoke_mkdir_basic-<timestamp>/` | Currently expected to fail with EPERM until fixed |
| `smoke_nested_directories.sh` | Nested a/b/c/d/e plus file remount verify | nested directories and file persist | directory creation/nesting blocked | full path, depth, tree before/after | `artifacts/smoke_nested_directories-<timestamp>/` | Blocked until mkdir passes |
| `smoke_directory_many_entries_small.sh` | 100 files in one directory | count/sample hashes verify after remount | many directory entries blocked | expected/actual count, sample hashes | `artifacts/smoke_directory_many_entries_small-<timestamp>/` | Blocked until mkdir passes |
| `smoke_sync_pressure_small.sh` | 100 small write/sync operations | no corruption or dmesg danger | sync/writeback issue | count, total bytes, dmesg danger scan | `artifacts/smoke_sync_pressure_small-<timestamp>/` | Blocked until mkdir and file paths are clean |
| `smoke_repeated_mount_unmount_10.sh` | Mount/unmount same image ten times | all cycles clean | mount/loop/ref cleanup issue | cycle pass/fail, refcount, loop state | `artifacts/smoke_repeated_mount_unmount_10-<timestamp>/` | Expected only from clean state; exposes teardown issues |
| `smoke_module_unload_after_filesystem_use.sh` | Unload after real filesystem use | rmmod succeeds; no ktxnmgrd/entd | module reference or daemon stuck | lsmod after rmmod, ktxnmgrd/entd checks | `artifacts/smoke_module_unload_after_filesystem_use-<timestamp>/` | May expose stuck module/ktxnmgrd |
| `smoke_failed_operation_teardown.sh` | Expected failed op then cleanup | failure does not poison module/loop/thread cleanup | failed mkdir/nonexistent read leaves dirty state | deliberate failure, cleanup result, rmmod result | `artifacts/smoke_failed_operation_teardown-<timestamp>/` | Failed mkdir may expose stuck module/ktxnmgrd |
| `smoke_dmesg_cleanliness.sh` | Scan dmesg for danger terms | no BUG/Oops/panic/WARNING/etc. | kernel warning/error appeared | full dmesg and filtered dmesg | `artifacts/smoke_dmesg_cleanliness-<timestamp>/` | Not clean until all required tests run without danger |
| `smoke_fsck_after_clean_unmount.sh` | fsck.reiser4 after clean unmount | fsck clean/acceptable exit | on-disk consistency concern | fsck version, full output, exit code | `artifacts/smoke_fsck_after_clean_unmount-<timestamp>/` | Not READY_TO_TRY until pass |
| `smoke_v3_short_stress.sh` | Short V3 workload: nested dirs, 500 files, rename/delete, remount verify | operation counts and remount verification pass | stress workload blocked | operation counts, hashes, remount verification, dmesg scan | `artifacts/smoke_v3_short_stress-<timestamp>/` | Blocked until mkdir/teardown are clean |
| `smoke_v3_repeat_from_clean_boot.sh` | Post-reboot repeat of V3 validation | boot recorded, repeat stress passes, final clean state | not clean after reboot or stress blocked | boot time, git, kernel, summaries, final state | `artifacts/smoke_v3_repeat_from_clean_boot-<timestamp>/` | Final gate; V3 personal status not READY_TO_TRY until pass |

Every V3 smoke test creates `summary.txt`, `state-before.txt`, `state-after.txt`, `state-after-cleanup.txt`, `dmesg-before.txt`, `dmesg-after.txt`, `dmesg-filtered.txt`, and `command-log.txt` when practical. Current known blockers remain honest: `mkdir` is expected to fail with `EPERM` until fixed, and teardown after failed mkdir may expose stuck module references, `ktxnmgrd`, or loop devices.
