# Ergonomics First

Reiser4-NX is not being revived merely to become technically impressive.

It is being revived to become usable, lovable, intuitive, fun, and dangerously easy to build upon.

A filesystem should not feel like punishment.

The greatest systems in history were not always the most technically pure systems. They were the systems people wanted to return to.

Systems people missed when they disappeared.

Systems that reduced friction between human creativity and action.

The goal is not merely performance.

The goal is flow.

We want researchers, artists, programmers, archivists, modders, data hoarders, and ordinary users to feel like the filesystem is helping them think instead of resisting them.

A filesystem should make people say:

“Wait… this is easy.”

“I can get lost in my work here.”

“I actually enjoy using this.”

That means:

- ergonomic tooling
- intuitive behavior
- low-friction metadata systems
- discoverable structures
- graceful defaults
- powerful extensibility
- understandable recovery
- visible state
- composable workflows
- systems that invite experimentation instead of punishing it

The future of storage should not feel oppressively simple, or desparingly complex.

It should feel alive.

Technical greatness matters.

But if people do not enjoy inhabiting the system, eventually they leave it behind.

Reiser4-NX aims to become both:

- technically formidable
- deeply inhabitable
